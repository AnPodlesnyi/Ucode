CSS

Description

Cascading Style Sheets (CSS) is a style sheet language used for describing the presentation of a document written in a markup language such as HTML. CSS is a cornerstone technology of the World Wide Web, alongside HTML and JavaScript.
CSS is designed to enable the separation of presentation and content, including layout, colors, and fonts.
This separation can improve content accessibility, provide more flexibility and control in the specification of presentation characteristics, enable multiple web pages to share formatting by specifying the relevant CSS in a separate .css file which reduces complexity and repetition in the structural content as well as enabling the .css file to be cached to improve the page load speed between the pages that share the file and its formatting.

Code example

p {
font-family: arial, helvetica, sans-serif;
}
h2 {
font-size: 20pt;
color: red;
background: white;
}
.note {
color: red;
background-color: yellow;
font-weight: bold;
}
p#paragraph1 {
padding-left: 10px;
}
a:hover {
text-decoration: none;
}

Protocol

Description

The Internet Protocol (IP) is the network layer communications protocol in the Internet protocol suite for relaying datagrams across network boundaries. Its routing function enables internetworking, and essentially establishes the Internet. IP has the task of delivering packets from the source host to the destination host solely based on the IP addresses in the packet headers. For this purpose, IP defines packet structures that encapsulate the data to be delivered. It also defines addressing methods that are used to label the datagram with source and destination information.

What are the layers of the OSI model?

The Open Systems Interconnection (OSI) model is an abstract representation of how the Internet works. It contains 7 layers, with each layer representing a different category of networking functions.

DNS - describe all kind of records

  • A (address record) - An address record that indicates the correspondence between a domain name and an IP address (IPv4).
  • AAAA (IPv6 address record) - Similar to A record, but specifies a domain name match for IPv6.
  • CNAME (canonical name record) is a record that allows you to assign a canonical name to a domain for an alias (one-level redirection).
  • DKIM signature (DomainKeys Identified Mail) is an e-mail authentication technology that allows you to confirm the identity of the sender.
  • MX (mail exchange) - a record pointing to the address of the mail gateway for the domain.
  • NS (name server) - indicates the DNS server serving this domain, i.e. indicates the servers to which the domain is delegated.
  • PTR (pointer) is a record that "links" an IP address to a domain name.
  • SOA (Start of Authority) - the record describes the main / initial settings of the zone, defines the zone of responsibility of this server.
  • An SPF record (sender policy framework) is used to protect a domain from forgery.
  • SRV (server selection) - this record type indicates the servers that provide the operation of certain services in this domain (for example, Jabber).
  • TXT - record contains auxiliary information about the domain (record of arbitrary binary data).
